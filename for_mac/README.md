# 画面キャプチャツール (macOS版) の使い方

本ツールは、試験中の画面をバックグラウンドで定期的に記録するためのツールです。

## ⚠️ 初回起動時の重要ステップ (必須)

macOSのセキュリティ保護機能（Gatekeeper）により、USBメモリ等からコピーしたアプリはそのままでは開けない場合があります。以下の手順で起動してください。

1.  **「1_開始」を右クリック（またはControlキーを押しながらクリック）**します。
2.  メニューから**「開く」**を選択します。
3.  「開発元を確認できませんが、開きますか？」という確認が出たら、再度**「開く」**をクリックします。
    *   ※この操作は各アプリ（1〜3）の初回起動時のみ必要です。

---

## 🚀 操作手順

### 1. 試験を開始する
1.  **「1_開始」**アプリを起動します。
2.  学籍番号を入力し、「開始」を押します。
3.  画面の案内に従い、**「画面収録」の許可設定**を行ってください。
    *   システム設定が開いたら、このアプリのスイッチを「オン」にします。
4.  設定完了後、アプリのダイアログで**「準備OK」**を押すと、記録が開始されます。

### 2. 動作を確認する
*   試験中、正しく記録されているか不安な場合は**「2_状況確認」**アプリを起動してください。
*   「正常に稼働中（最終撮影: 〇時〇分）」と表示されれば問題ありません。

### 3. 試験を終了し、データを提出する
1.  試験監督の指示に従い、**「3_提出と停止」**アプリを起動します。
2.  記録が停止し、デスクトップに **`{学籍番号}_evidence.zip`** というファイルが作成されます。
3.  このZIPファイルを、指定された提出先（LMS等）にアップロードしてください。

---

## 🛠 トラブルシューティング

*   **画面収録の許可ポップアップが出ない場合**
    *   「システム設定」＞「プライバシーとセキュリティ」＞「画面収録」を開き、手動でアプリを一覧に追加してオンにしてください。既に一覧にある場合は、一度「ー」で削除してから再度追加（ドラッグ＆ドロップ）すると解決する場合があります。
*   **アプリが動かなくなった場合**
    *   一度「3_提出と停止」を実行してから、再度「1_開始」を試してください。
