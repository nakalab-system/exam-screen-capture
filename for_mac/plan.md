# Mac版 試験画面キャプチャツール 仕様書

## 1. ツールの概要
本ツールは、BYOD（学生個人の持ち込みPC）および完全オフライン環境での試験において、試験の公正性を保つために画面をバックグラウンドで定期的にキャプチャ（静止画記録）し、証拠データを生成するシステムである。
既存のWindows版（PowerShellベース）の仕様を継承しつつ、macOSの標準機能のみを用いて実装する。追加ソフトウェアのインストールは不要とし、USBメモリ経由での配布を想定する。

---

## 2. システム要件と前提条件
* **対象OS:** macOS (Catalina以降。プライバシー権限TCCの仕様に準拠)
* **実行環境:** macOS標準搭載の機能（ShellScript, AppleScript, osascript, screencapture等）のみを使用。
* **ネットワーク:** 完全オフライン環境で動作すること。
* **配布方法:** USBメモリ（exFAT/FAT32フォーマットを想定）。実行権限消失を避けるため `.app` 形式でパッケージングし、`.dmg` または `.zip` に格納して配布する。

---

## 3. アプリケーション構成と動作フロー
学生が直感的に操作できるよう、3つの `.app` ファイルを提供する。UI（ダイアログ）は `osascript` を用いてMac標準のポップアップで実装する。

### 3-1. 1_開始.app (試験開始)
* 学生に「学籍番号」の入力を求めるダイアログを表示する。
* `~/Library/Application Support/CaptureSystem` （不可視領域）に一時フォルダを作成し、学籍番号を保存する。
* フォルダに対して `chflags uchg` を実行し、学生が誤って削除できないようロックをかける。
* バックグラウンドキャプチャプロセス（`capture.sh`）を非表示で起動する。
* **[初回のみ]** 画面録画権限（TCC）がない場合は、システム設定を自動で開き、許可を促す案内を表示する。

### 3-2. 2_状況確認.app (ステータス確認)
* バックグラウンドのキャプチャプロセス（`capture.sh`）が正常に稼働しているか、プロセスID（PID）等から判定する。
* 正常稼働している場合、保存先フォルダ内の最新画像のタイムスタンプを取得し、「正常に稼働中（最終撮影: XX時XX分）」とポップアップ表示する。

### 3-3. 3_提出と停止.app (試験終了)
* バックグラウンドのプロセスを安全に強制終了（`kill`）する。
* フォルダのロックを `chflags nouchg` で解除する。
* デスクトップに `{学籍番号}_evidence.zip` としてアーカイブ（`zip`）を作成する。
* 一時保存用フォルダ内のデータを完全に削除（クリーンアップ）し、完了メッセージを表示する。

## 4. 技術仕様（コアプロセス：capture.sh）
バックグラウンドで稼働するシェルスクリプトの要件は以下の通り。

| 項目 | 実装方針・使用コマンド |
| :--- | :--- |
| **一時保存先** | `~/Library/Application Support/CaptureSystem` |
| **キャプチャ間隔** | `sleep $((RANDOM % 61 + 30))` (30〜90秒のランダム間隔) |
| **キャプチャ処理** | `screencapture -m -x -t jpg <ファイルパス>` |
| **画面指定** | `-m` オプションにより、メインディスプレイのみを対象とする |
| **音の抑制** | `-x` オプションによりシャッター音を無効化する |
| **スリープ防止** | 稼働中は `caffeinate -d` を併用し、Macの自動スリープを防止する |
| **プロセス管理** | 起動時に自身のPIDをテキストに書き出し、二重起動防止と終了時の正確な `kill` を担保する |

## 5. Mac（BYOD/USB配布）特有の懸念点と対策（実装要件）

1. **Gatekeeperによる起動ブロック対策**
   * **課題:** USBからコピーした未署名の `.app` は「開発元が未確認」としてブロックされる。
   * **対策（運用）:** ツール実行マニュアルに「初回起動時はアイコンを右クリック（またはControl+クリック）し、『開く』を選択する」手順を明記する。

2. **ゾンビプロセス・二重起動の防止**
   * **課題:** 学生が `1_開始.app` を連打すると、キャプチャプロセスが複数立ち上がり、負荷増大やファイル競合を引き起こす。
   * **対策（実装）:** 起動時にPIDチェックを行い、すでに実行中の場合は「すでに試験は開始されています」と警告を出し、処理を中断させる。

3. **試験中の自動スリープによる録画停止**
   * **課題:** 無操作状態が続くとMacがスリープし、シェルスクリプトが停止する。
   * **対策（実装）:** `capture.sh` を `caffeinate` コマンド配下で実行し、強制的に起きている状態を維持する。

4. **マルチディスプレイ（デュアルモニター）環境**
   * **課題:** デフォルトの `screencapture` は全画面を分割保存しようとし、エラーの原因になる。
   * **対策（実装）:** 前述の通り `-m` オプションを必須とし、メイン画面のみを確実に取得する。

5. **USB（exFAT等）配布における権限消失対策**
   * **課題:** exFAT形式のUSBではファイルシステム上、Unixの実行権限（`+x`）が保持できないため `.command` スクリプトが動かなくなる。
   * **対策（実装）:** スクリプトをAppleScript等でラップして `.app` バンドル化し、さらに `.dmg` にパッキングして配布することで権限問題を回避する。
